Motivation & Problem Statement

Emergency situations — cardiac events, accidental falls, personal safety threats — demand immediate response. In practice, victims are often unable to call for help: the phone is locked, hands are occupied, or panic impairs fine motor control.

Existing wearable solutions suffer from critical limitations:

  • Require manual interaction at the moment of crisis
  • Depend on smartphone connectivity (Bluetooth/Wi-Fi)
  • Offer only single-modality detection — no redundancy
  • Cloud-dependent voice recognition with high latency
Key Insight: A victim experiencing cardiac arrest, collapse, or physical assault may be completely unable to initiate any deliberate action. The system must detect the emergency autonomously.

The Smart Safety Band addresses this with independent, redundant SOS trigger mechanisms operating directly on the device — requiring zero smartphone dependency.

System Overview

The band is a self-contained wearable that continuously monitors the user and autonomously dispatches SOS alerts with real-time GPS coordinates over the GSM cellular network.

SOS Trigger Mechanisms

The band implements two direct SOS trigger pathways plus an autonomous detection mechanism. Any single trigger is sufficient to dispatch an alert, ensuring broad coverage across emergency scenarios.

Trigger 1  Manual Touch Sensor

1
User applies deliberate pressure to the capacitive touch sensor on the band surface.
2
GPIO interrupt fires immediately — no polling delay.
3
SOS dispatched instantly with current GPS coordinates.
Best for: conscious victim who can act but cannot dial a phone. Zero processing latency.

Trigger 2  Voice Trigger via VC02

The VC02 dedicated voice recognition chip runs entirely on-device with no cloud dependency. It continuously listens for pre-programmed distress keywords.

1
VC02 chip performs local acoustic pattern matching against stored keyword models.
2
On keyword detection, GPIO signal fires to the A9G firmware.
3
SOS dispatched immediately — victim need not touch the device at all.
"HELP" "SOS" "EMERGENCY" — configurable keywords
Why VC02 over cloud ASR? Offline processing eliminates network latency and ensures the trigger works without internet — critical in disaster scenarios where networks are congested.
✦ Novelty — Autonomous Freeze Detection with Haptic Confirmation

The most novel aspect of the Smart Safety Band is its autonomous freeze detection mechanism — a two-stage algorithm that identifies medical emergencies without any user action, while using a coin vibrator as a confirmation step to prevent false alarms.

Fig. 2 — Freeze detection: two-condition check → coin vibrator alert → confirmation window → SOS or cancel

Why this is novel: Unlike existing wearables that fire alerts immediately on threshold breach, this system introduces a human-in-the-loop confirmation step via haptic feedback. The coin vibrator alerts the victim — if conscious and safe, they can cancel within the time window. If no response is received, the system concludes an emergency and dispatches SOS autonomously. This drastically reduces false positives while preserving fully autonomous detection.
  • Elevated HR alone (e.g. exercise) does not trigger — AND gate prevents false alarms
  • Zero movement alone (e.g. sleep) does not trigger — both conditions must be true
  • Coin vibrator gives the victim a chance to self-cancel before SOS fires
  • Entire mechanism runs offline — no cloud, no smartphone required
Software Architecture

The full stack spans three layers: embedded firmware on the A9G, a backend API server, and a web frontend dashboard.

Alert Dispatch Pipeline

Once any trigger fires, the firmware executes the following pipeline deterministically:

1
GPS fix acquired — A9G queries GNSS module, obtains lat/lng coordinates (~5 m accuracy)
2
SMS sent via GSM — pre-formatted message with coordinates dispatched to all stored contacts
3
HTTP POST over GPRS — structured JSON payload sent to backend with timestamp, coordinates, trigger type
4
Backend stores event — alert logged to database, push notification sent to registered contacts
5
Frontend updates — live map pin placed, alert card shown in dashboard with full details
Total pipeline latency target: < 5 seconds from trigger event to SMS delivery under normal GSM coverage.
